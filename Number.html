parseInt e parseFloat

parseInt() converte un valore in un numero intero (senza decimali); Può contenere due argomenti; il primo è il valore da analizzare ed il secondo è una radice. La radice serve a specificare quale sistema numerico da usare. Conviene sempre specificare tale valore per evitare di avere differenti risultati interpretativi dai vari browser. 

parseInt(string, radix);
La funzione parseInt converte il primo argomento ad una stringa, lo analizza e restituisce un numero intero oppure NaN. In caso di NaN il valore restituito sarà la rappresentazione dell'intero decimale del primo argomento preso come numero. Per esempio una radice di 10 indica di convertire a numero decimale, radice 8 a ottale, 16 a esadecimale e cosi via. 
parseInt("100px");			// 100
parseInt("123");			// 123 (implicit decimal)
parseInt("123hello");			// 123 (implicit decimal)
parseInt(220.84);			// 220
parseInt(0.0000001);			// 1
parseInt("010");			// 8 (implicit octal)
parseInt("0xCAFE");			// 51966 (implicit hexadecimal)
parseInt("10.20Hello");			// 10 (vedi la differenza con 'parseFloat')

Con secondo parametro
parseInt("010", 10);			// 10 (explicit decimal)
parseInt("11", 2);			// 3 (explicit binary)
parseInt("10.20Hello", 10);		// 10 (vedi la differenza con 'parseFloat')
parseFloat, a differenza di parseInt converte un valore sempre in un numero decimale 
parseFloat('421.45');			// 421.45
parseFloat('+421.45');			// 421.45
parseFloat('-421.45');			// -421.45
parseFloat("10.20Hello");		// 10.2 (qui il risultato è diverso che con 'parseInt')
Un'altro modo per convertire una stringa in numero è con l'utilizzo degli operatori unari + e - 
+ "42"					// restituisce 42 
(+"1.1") + (+"1.1")			// restituisce 2.2
altrimenti senza operatore unario
"1.1" + "1.1"				// restituisce "1.11.1"

Numbers to Strings

Quando si unisce un numero ad una stringa il risultato è sempre una stringa 

console.log("La somma è  " + 1 + 2);			// restituisce una stringa => "La somma è  12"
console.log("OK " + 0.0000001);				// "OK 1e-7"

Nel seguente facendso uso dell'operatore + il discorso viene cambiato
console.log("La somma è " + (1 + 2) );			// restituisce la stringa "La somma è 3"

valueOf

Il metodo valueOf ottiene il primitivo valore di un oggetto numero come tipo di numero 

var myNumero = new Number(24);
var myN = myNumero.valueOf();
console.log(myN);			// 24

isNaN( )

Serva a verificare se un valore non sia un numero. Succede che il valore passato viene convertito in un numero e quindi verifica se sia un numero e restituisce un risultato. Siccome la sua gestione è leggermante confusionale è conveniente far affidamento su typeof 

isNaN("testo");				// true 
isNaN(37);				// false         
isNaN("37");				// false (37 è convertito a numero e quindi valutato 'false' perchè è un numero) 
isNaN(null);				// false       
isNaN("");				// false (la stringa vuota è convertita a zero che è un numero )  
isNaN(" ");				// false (la stringa vuota con spazio è convertita a zero che è NaN)  

isNaN(new Date());			// false                 
isNaN(new Date().toString());		// true

Verifica numeri interi ( Number.isInteger() )

Number.isInteger() è una nuova funzione introdotta da ECMAscript 6. Nelle versioni precedenti non c'è un metodo per stabilire con esattezza se un numero sia intero o meno. Comunque, al momento, ci sono delle soluzioni abbastanza semplici per ovviare al problema 

function isInteger(x) {
	return Math.round(x) === x;
}

console.log(isInteger(45));		true
console.log(isInteger(45.4));		false
console.log(isInteger('45'));		false
Invece di Math.round() si potrebbero benissimo usare anche i metodi Math.ceil() e Math.floor()
Altra funzione alternativa, che trovo più completa, è la seguente 
function isInteger(x) {
	return (typeof x === 'number') && (x % 1 === 0);
}
