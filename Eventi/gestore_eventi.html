Gestore di eventi

Il modo più semplice per eseguire del codice JavaScript in risposta ad un evento consiste nell'utilizzare un gestore di eventioevent handler
elemento.onevento = eventoHandler;

Esempio:
La prima lettera dell'alfabeto è la A
var link = document.getElementById("lettera");
link.onclick = wiki_link.clickHandler; {
clickHandler: function() {
alert("OK");
}
Ma c'è un problema da tener conto, che non è possibile assegnare all'elemento un gestore di eventi finchè l'elemento non è stato caricato
Il modo per scrivere del codice dopo che è avvenuto il caricamento del documento è :
var wiki_link = {
init: function() {
var link = document.getElementById("lettera");
link.onclick = wiki_link.clickHandler; },
clickHandler: function() {
alert("OK"); }
};
Core.start(wiki_link);
Quindi quando si clikkerà sul link A si aprirà una finestra (alert) che darà il messaggio di "OK";

Problema con i gestori di eventi

E' possibile assegnare un gestore di eventi soltanto ad un determinato evento su un determinato elemento

Se per esempio abbiamo:
elemento.onclick = script01.clickHandler;
elemento.onclick = script02.clickHandler;
quando si verifica un clic su elemento verrà eseguito solo il clickHandler dello script 02.
Ci sarebbero soluzioni ad aggirare il problema creando funzioni, ma alla fine il lavoro risulterebbe eccessivo. Allora rivolgersi ai Listener per eventi
